01 Decision
Decision

A virtual HyperDeck that types into a local player.

One cross-platform Go binary emulates a single Blackmagic HyperDeck on TCP 9993. It locks onto one running player (VLC, Example Player, or Mitti) and translates HyperDeck transport commands into keystrokes. A tray icon shows whether it is locked on. The only OS-specific code lives behind one Injector interface, so nearly everything is built and tested on macOS/Linux against a mock.

#DecisionChoiceWhy
1Structure Hexagonal Pure domain core; network, OS, UI are swappable ports & adapters. Core tests everywhere.
2Platform scope Win + macOS Mitti is macOS-only; VLC/the player also run there. Mock adapter covers Linux/CI.
3Deck count One app, one deck One player runs at a time; profile auto-selected by window match.
4Injection Per-profile focus (foreground + real keys) or background (PostMessage).
5State feedback Modeled + detect Open-loop model corrected by best-effort title parsing.
6Config YAML profiles Adding an app is config, not code.
ATEM and hardware remotes poll device info and slot status and may reject a deck that answers wrong — protocol fidelity is a first-order requirement, not a nicety.
Architecture

A pure hexagon, all dependencies pointing inward.

The virtual deck is a pure domain core with no imports of any network, OS, filesystem, or UI package — and no build tags. Adapters on the outside translate between the core and the world, depending on ports the core defines. That one rule is what lets the entire core compile and test on any OS. 

            DRIVING SIDE                 CORE (hexagon)              DRIVEN SIDE
   HyperDeck controller ─▶ hyperdeck ─▶┃ Transport / Query ┃─▶ KeyInjector  ─▶ injector ─▶ player
        tray menu clicks ─▶ tray ──────▶┃   VirtualDeck     ┃─▶ WindowEnum.  ─▶ injector
                                        ┃   LockManager     ┃─▶ ClipSource   ─▶ clipsource

Ports

The core's contracts, in two directions.

Driving ports are the deck's API, called inward by driving adapters. Driven ports are what the deck needs from the world, implemented outward by driven adapters. The core owns both interface definitions.

Judicious, not dogmatic: coarse ports (not one-per-method), domain types are the contracts (no DTO layer), Clock exists only for test determinism, tray is one bidirectional adapter, and window matching stays in the core — only enumeration crosses a port.
Driven adapter: injector

One driven port isolates every platform call.

KeyInjector and WindowEnumerator are the single OS seam. The interfaces live in core/port; three build-tagged adapters implement them. Tests use the mock. 

// core/port — defined by the core, implemented by driven adapters.
type Window struct {
    Handle  uintptr
    Title   string
    Process string
}

type KeyInjector interface {
    Focus(w Window) error                  // foreground the window (focus mode)
    SendKeys(w Window, keys []Chord) error // e.g. ctrl+right, cmd+esc, space
}

type WindowEnumerator interface {
    OpenWindows() ([]Window, error)        // LockManager matches these to profiles
}
windowsSendInput for focus mode; PostMessage for background; EnumWindows + GetWindowThreadProcessId to enumerate. No cgo.
darwinCGEventPost for keys; Accessibility / AppleScript activate for focus. Needs cgo + the Accessibility permission.
mockRecords every call; scriptable to simulate find/focus success or failure. Drives all CI tests.
Background PostMessage is silently dropped by DirectInput and fullscreen apps. Default uncertain profiles to focus.
State & flow

Open-loop state makes toggle keys safe.

We cannot read a player's true play state, so the deck tracks a modeled state and corrects it with best-effort detection. That model is what lets us serve HyperDeck's discrete play/stop on top of apps like Example Player, where Space only toggles. 

hyperdeck adapter --Transport port--> VirtualDeck.Play()
                                              |
                                              v
                                  resolve chords --> KeyInjector port --> injector --> player
Reconciler (Clock tick) --> StateProbe port --> reconcile modeled state
                        <-- Query port (transport info / notify) <-- VirtualDeck
LockManager (Clock tick) --> WindowEnumerator port --> lock / unlock --> StatusPresenter

Toggle resolution (inside the core)

func (d *VirtualDeck) apply(cmd Command) []Chord {
    // For a toggle key, emit only when the requested state differs from the
    // modeled state. play-while-playing emits nothing; stop-while-stopped too.
    if d.profile.IsToggle(cmd.Key) {
        if d.state == cmd.WantState {
            return nil // already there — sending Space would flip it wrong
        }
        d.state = cmd.WantState
        return []Chord{d.profile.Keymap[cmd.Key]}
    }
    // Discrete keys (VLC stop, Mitti panic) just fire and set the model.
    d.state = cmd.WantState
    return []Chord{d.profile.Keymap[cmd.Key]}
}
Profiles

Each app is a YAML profile.

A profile declares its window match, keymap, injection mode, clip source, state-detection rule, and homing sequence. New apps need no code. 

profiles:
  - id: vlc
    match: { process: ["vlc.exe", "VLC"], title_regex: "VLC media player" }
    injection: background
    keymap: { play: "space", stop: "s", next: "n", prev: "p" }
    toggle_keys: []                       # discrete keys
    clip_source: { type: playlist_file, path: "${VLC_PLAYLIST}" }
    state: { type: title_regex, playing: ".+ - VLC" }
    homing: ["s"]                         # run on explicit Re-home only

  - id: example_player
    match: { process: ["Example Player"] }
    injection: focus
    keymap: { play: "space", next: "ctrl+right", prev: "ctrl+left" }
    toggle_keys: ["space"]                # Space toggles -> model decides
    clip_source: { type: positional, count: 50 }
    state: { type: none }
    homing: []

  - id: mitti
    match: { process: ["Mitti"] }
    injection: focus                      # macOS CGEvent
    keymap: { play: "enter", pause: "space", stop: "cmd+esc",
              next: "cmd+down", prev: "cmd+up" }
    clip_source: { type: mitti }
    state: { type: none }
    homing: []
locator binds the first profile whose match succeeds. Top-level config holds bind address/port, poll interval, and log paths.
Protocol surface

What the deck answers, and how it maps.

The hyperdeck driving adapter owns the wire format and calls the Transport/Query ports — it never touches the OS. Text over TCP 9993. Simple commands ack with 200 ok; responses use a 2xx code with a colon-terminated body; errors use 1xx; async notices use 5xx. Exact grammar is mined from the HyperDeck product manual during planning.

CommandMaps toNotes
play / play:play key (toggle-aware)Speed params best-effort; no jog/shuttle in MVP.
stopstop key (or toggle)Models stopped state.
recordrecord key if definedAcked as no-op when a profile has none.
goto: clip id: Ndelta of next/prev keys1-based ids; delta from modeled index.
clips getclipsource.List()VLC real names; the player positional; Mitti best-effort.
transport infomodeled + detected statePolled ~1s; primary controller status feed.
slot infolock stateOne present slot when locked; empty when not.
device info / notify / pingidentity / asyncFidelity for ATEM & hardware remotes.
Unsupported commands return a clean protocol error — never drop the connection.

Operations

Build, run, recover.

Cross-compilePure-Go Windows target builds from macOS/Linux. macOS target needs cgo, so build on a Mac (or macOS CI runner).
Tray statusLocked = connected icon + present slot/clips. Lost lock = disconnected icon; deck reports no remote/slot.
Re-homeTray action runs the profile's homing sequence and resets modeled state — the recovery path after drift.
macOS permissionSynthetic input needs Accessibility. Detect absence and guide the user; fail loudly, not silently.
DiscoveryIP-only for MVP — Companion/ATEM add the deck by address. mDNS advertise is a later enhancement.
Config & logsYAML at an OS-standard path; invalid config fails fast at startup naming the profile and field.
Risks & confidence

Where this could bite.

ATEM / hardware fidelityMay reject a deck that mis-answers identity/status. Mitigate: implement the manual's command set faithfully; test against ATEM Software Control early.
Background injection limitsPostMessage ignored by DirectInput/fullscreen. Mitigate: per-profile mode, default focus.
Mitti clip namesProprietary playlist + native HyperDeck emulation already. Clip-name reporting is best-effort; discrete keys work fine.
macOS permissionsAccessibility required for input and window introspection; absence must be detected and surfaced.
